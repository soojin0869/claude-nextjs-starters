# 노션 기반 견적서 웹 뷰어 MVP PRD

## 핵심 정보

**목적**: 노션 데이터베이스에 입력한 견적서를 클라이언트가 별도 계정 없이 공유 URL 하나로 확인하고 PDF로 저장할 수 있게 한다.
**사용자**: 프리랜서 및 1인 사업자(어드민)와 그 클라이언트(수신자)

---

## 사용자 여정

```
[어드민]
1. 노션 데이터베이스에 견적서 항목 입력
   ↓ 자동으로 고유 견적서 ID 생성됨 (Notion 페이지 ID)

2. 생성된 공유 URL 복사
   예: https://your-domain.com/invoice/[notion-page-id]
   ↓ 클라이언트에게 URL 전달 (이메일, 카카오톡 등)

[클라이언트]
3. 공유 URL 브라우저에서 열기
   ↓ Notion API로 견적서 데이터 조회

   [견적서 존재] → 견적서 상세 페이지 표시
   [견적서 없음] → 404 에러 페이지 표시
   ↓

4. 견적서 상세 페이지에서 내용 확인
   ↓ PDF 다운로드 버튼 클릭

5. 브라우저에서 PDF 생성 및 다운로드 완료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명             | 설명                                                          | MVP 필수 이유                                           | 관련 페이지            |
| -------- | ------------------ | ------------------------------------------------------------- | ------------------------------------------------------- | ---------------------- |
| **F001** | 견적서 데이터 조회 | Notion API로 특정 페이지 ID의 견적서 데이터를 읽어온다        | 서비스의 유일한 데이터 소스                             | 견적서 상세 페이지     |
| **F002** | 견적서 웹 렌더링   | 조회한 데이터를 시각적으로 읽기 좋은 견적서 형태로 표시한다   | 클라이언트가 내용을 확인하는 핵심 기능                  | 견적서 상세 페이지     |
| **F003** | PDF 다운로드       | 화면에 표시된 견적서를 PDF 파일로 생성해 로컬에 저장한다      | 공식 문서로 보관하려는 클라이언트의 핵심 니즈           | 견적서 상세 페이지     |
| **F004** | 견적서 URL 생성    | Notion 페이지 ID 기반의 공유 가능한 URL을 어드민에게 제공한다 | 어드민이 클라이언트에게 견적서를 전달하는 유일한 수단   | 어드민 대시보드 페이지 |
| **F005** | 견적서 목록 조회   | Notion 데이터베이스의 전체 견적서 목록을 조회·관리한다        | 어드민이 발행한 견적서를 한눈에 파악하기 위한 필수 기능 | 어드민 대시보드 페이지 |

### 2. MVP 필수 지원 기능

| ID       | 기능명          | 설명                                                     | MVP 필수 이유                                  | 관련 페이지                                  |
| -------- | --------------- | -------------------------------------------------------- | ---------------------------------------------- | -------------------------------------------- |
| **F010** | 어드민 인증     | 환경변수 기반 패스워드로 어드민 대시보드 접근을 제한한다 | 견적서 목록 및 URL이 외부에 노출되지 않아야 함 | 어드민 로그인 페이지, 어드민 대시보드 페이지 |
| **F011** | 404 / 에러 처리 | 존재하지 않는 견적서 ID 접근 시 안내 페이지를 표시한다   | 잘못된 URL 접근에 대한 최소한의 UX 처리        | 에러 페이지                                  |

- 견적서 상태 관리 (발송됨, 승인됨, 거절됨)
- 클라이언트 서명 기능
- 견적서 이메일 자동 발송
- 견적서 만료일 설정 및 접근 차단
- 다국어 지원

## 노션 데이터베이스 스키마

어드민이 Notion에서 생성해야 하는 **견적서 데이터베이스(Invoice Database)** 프로퍼티 정의입니다.

### 상위 데이터베이스: Invoice (견적서)

| 프로퍼티명     | Notion 타입 | 설명                                          | 필수 여부 |
| -------------- | ----------- | --------------------------------------------- | --------- |
| `title`        | Title       | 견적서 제목 (예: "2026년 5월 웹 개발 견적서") | 필수      |
| `client_name`  | Rich Text   | 수신 클라이언트명 (개인 또는 회사명)          | 필수      |
| `client_email` | Email       | 클라이언트 이메일                             | 선택      |
| `issue_date`   | Date        | 견적서 발행일                                 | 필수      |
| `due_date`     | Date        | 견적 유효기간                                 | 선택      |
| `status`       | Select      | 견적 상태: `초안` / `발송됨` / `확정됨`       | 필수      |
| `note`         | Rich Text   | 하단 메모 또는 결제 안내                      | 선택      |
| `currency`     | Select      | 통화 단위: `KRW` / `USD`                      | 필수      |

### 하위 항목: Invoice Item (견적 항목)

> Notion에서 별도 데이터베이스로 구성하거나, 상위 Invoice 페이지의 본문(Blocks)으로 테이블을 직접 작성하는 방식 중 선택 가능. **MVP에서는 본문 테이블 방식을 권장** (구현 복잡도 최소화).

Invoice 페이지 본문에 다음 컬럼을 가진 Simple Table 블록 작성:

| 컬럼명   | 설명                              |
| -------- | --------------------------------- |
| `항목명` | 서비스 또는 작업 이름             |
| `수량`   | 수량 (숫자)                       |
| `단가`   | 단가 (숫자)                       |
| `금액`   | 수량 × 단가 (수동 입력 또는 계산) |

### 어드민 정보: Notion 환경변수로 관리

별도 테이블 없이 환경변수로 관리:

| 환경변수               | 설명                           |
| ---------------------- | ------------------------------ |
| `NOTION_API_KEY`       | Notion Integration 토큰        |
| `NOTION_DATABASE_ID`   | 견적서 데이터베이스 ID         |
| `ADMIN_NAME`           | 발행자 이름 (견적서 하단 표시) |
| `ADMIN_EMAIL`          | 발행자 이메일                  |
| `ADMIN_PHONE`          | 발행자 연락처                  |
| `ADMIN_PASSWORD`       | 어드민 대시보드 접근 패스워드  |
| `NEXT_PUBLIC_BASE_URL` | 배포 도메인 (URL 생성용)       |

## API 설계 (Next.js App Router 기준)

### Route Handlers

| 엔드포인트                | 메서드 | 설명                                 | 인증        |
| ------------------------- | ------ | ------------------------------------ | ----------- |
| `GET /api/invoice/[id]`   | GET    | Notion API로 특정 견적서 데이터 조회 | 불필요      |
| `GET /api/admin/invoices` | GET    | Notion DB 전체 견적서 목록 조회      | 어드민 쿠키 |

### Server Actions

| 액션명              | 설명                                | 인증   |
| ------------------- | ----------------------------------- | ------ |
| `adminLoginAction`  | 패스워드 검증 후 httpOnly 쿠키 발급 | 불필요 |
| `adminLogoutAction` | 어드민 쿠키 삭제                    | 불필요 |

### 페이지 (App Router)

| 경로            | 렌더링 방식                 | 설명               |
| --------------- | --------------------------- | ------------------ |
| `/invoice/[id]` | SSR (generateMetadata 포함) | 견적서 상세 (공개) |
| `/admin`        | SSR + 미들웨어 인증         | 어드민 대시보드    |
| `/admin/login`  | CSR                         | 어드민 로그인      |

### 미들웨어 (`middleware.ts`)

```
/admin 경로 접근 시:
  → 어드민 쿠키 없음 → /admin/login 리디렉션
  → 어드민 쿠키 있음 → 통과
```

- **Zod** - 패스워드 스키마 검증

### 데이터 소스

- **@notionhq/client** - Notion API 클라이언트 (서버사이드 전용)

### PDF 생성

- **html2canvas** + **jspdf** - 렌더링된 DOM을 캡처해 PDF 생성 (브라우저 클라이언트 측)

### 인증

- **httpOnly 쿠키** - 별도 라이브러리 없이 Next.js Server Actions + 미들웨어로 처리

### 배포

- **Vercel** - Next.js 15 최적화 플랫폼

### 패키지 관리

- **npm** - 의존성 관리
